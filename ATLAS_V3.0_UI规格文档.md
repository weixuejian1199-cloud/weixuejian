# ATLAS V3.0 UI 规格文档

> 本文档为 ATLAS V3.0 前端界面的完整设计规格，涵盖所有界面状态、交互逻辑、组件规范和响应式适配方案。所有设计决策均经过讨论确认，作为开发实施的唯一依据。

---

## 一、设计原则

ATLAS V3.0 的 UI 设计以 **Google Gemini 的极简风格** 为基准，结合 **Kimi 的文件管理交互**，在此基础上融入 ATLAS 自身的数据处理特色。核心原则如下：

**极简克制**：页面元素越少越好，每个元素必须有存在的理由。不放空壳功能入口，不做装饰性设计。

**对话驱动**：所有功能通过对话流触发和交付，模板库、HR 中心等功能入口以快捷标签形式嵌入首页对话框下方，不单独占据导航位置。

**数据专业**：在 Gemini 的通用对话风格基础上，强化数据展示能力——关键数字高亮、重型表格摘要卡片、文件处理状态可视化。

**浅色主题**：全局采用浅色主题，不提供深色模式切换。背景色使用浅灰（接近 Gemini 的 #f0f4f9），文字使用深灰/黑色，确保长时间使用不疲劳。

## 二、全局布局结构

### 2.1 桌面端布局

整体采用 **两栏结构**：左侧可折叠导航栏 + 右侧主内容区。不设常驻右侧分析面板。

```
┌──────────────────────────────────────────────────┐
│  ATLAS（logo）    [对话标题]      📎  ⋮  [头像]  │  ← 顶栏
├────────┬─────────────────────────────────────────┤
│ ☰ 🔍  │                                         │
│ ✏️新建 │                                         │
│        │           主内容区                       │
│ 聊天   │      （首页 / 对话流）                    │
│ ├今天  │                                         │
│ ├最近  │                                         │
│ ├更早  │                                         │
│        │                                         │
│        │                                         │
│ 🎁分享 │                                         │
│ ⚙️设置 │                                         │
└────────┴─────────────────────────────────────────┘
```

### 2.2 手机端布局

手机端 **1:1 对标 Gemini 移动端风格**，全屏对话，不显示侧栏。

```
┌─────────────────────┐
│ ☰  ATLAS    📎 [头像]│  ← 顶栏（精简）
├─────────────────────┤
│                     │
│    主内容区          │
│  （首页 / 对话流）    │
│                     │
│                     │
│ ┌─────────────────┐ │
│ │  对话输入框       │ │  ← 底部固定
│ │  + 上传    发送 → │ │
│ └─────────────────┘ │
└─────────────────────┘
```

左上角 ☰ 点击后弹出侧栏覆盖层（overlay），覆盖整个屏幕，背景半透明遮罩，点击遮罩或返回关闭。

## 三、Logo 规则

整个页面 **只允许出现一次 ATLAS 品牌标识**，位置在顶栏左侧（侧栏展开时紧邻搜索图标右侧，侧栏收起时紧邻 ☰ 按钮右侧），与 Gemini 的"双子座"品牌名位置完全一致。

首页欢迎区域、对话流、侧栏内部、页面底部均不重复出现 logo。

## 四、侧栏（Sidebar）

### 4.1 展开状态

宽度：**200px**（比 Gemini 的 ~260px 窄，把空间让给对话区）。

| 区域 | 内容 | 说明 |
|------|------|------|
| 顶部 | ☰ 收起按钮 + 🔍 搜索图标 | 搜索点击后展开搜索框，支持搜索历史对话 |
| 功能区 | ✏️ 新建对话 | 唯一的功能按钮，点击创建新对话并跳转首页 |
| 历史区标题 | "聊天" | 灰色小字标题 |
| 历史列表 | 对话标题列表 | 按时间分组：今天 / 最近 7 天 / 更早 |
| 底部 | 🎁 分享好礼 | 点击弹出分享覆盖层 |
| 底部 | ⚙️ 设置 | 点击弹出设置覆盖层 |

历史对话标题超出 200px 宽度时，使用省略号截断。当前活跃对话高亮显示（左侧蓝色竖条 + 背景微灰）。

### 4.2 收起状态

宽度：**48px**，只显示图标按钮，纵向排列。

| 位置 | 图标 | 功能 |
|------|------|------|
| 顶部 | ☰ | 展开侧栏 |
| 第二个 | ✏️ | 新建对话 |
| 底部倒数第二 | 🎁 | 分享好礼 |
| 底部最后 | ⚙️ | 设置 |

中间大面积留空，保持极简。所有图标 hover 时显示 tooltip 文字提示。

### 4.3 展开/收起交互

点击 ☰ 按钮切换展开/收起状态，带 200ms 的横向滑动动画。主内容区宽度随之自适应调整。桌面端默认展开，窗口宽度小于 768px 时自动收起。

## 五、顶栏（TopBar）

### 5.1 桌面端顶栏

高度：**56px**，背景色与页面背景一致（#f0f4f9），**无底部分割线**，与内容区无缝衔接（对标 Gemini，顶栏和内容区融为一体，不做视觉分隔）。

```
┌──────────────────────────────────────────────────────┐
│  ATLAS     [对话标题（居中）]          📎(3)  ⋮  [Z] │
└──────────────────────────────────────────────────────┘
```

| 位置 | 元素 | 说明 |
|------|------|------|
| 左侧 | ATLAS 品牌名 | 唯一 logo 位置，字体加粗，品牌色 |
| 居中 | 对话标题 | 首页时不显示；对话中显示自动生成的标题 |
| 右侧 | 📎 回形针 + 数量角标 | 当前对话有文件时显示，悬浮预览文件列表，点击展开文件面板 |
| 右侧 | ⋮ 更多菜单 | 对话中显示，首页不显示 |
| 右侧 | 用户头像 | 圆形头像，点击弹出用户菜单 |

### 5.2 ⋮ 更多菜单内容

点击后弹出下拉菜单：

| 菜单项 | 功能 |
|--------|------|
| ✏️ 重命名 | 修改当前对话标题 |
| 📌 置顶 | 将对话置顶到历史列表顶部 |
| 🗑️ 删除 | 删除当前对话（二次确认） |

### 5.3 用户头像菜单

点击头像弹出下拉菜单：

| 菜单项 | 功能 |
|--------|------|
| 个人信息 | 查看/编辑昵称、头像 |
| 退出登录 | 退出当前账户 |

### 5.4 手机端顶栏

高度：**48px**，背景色与页面背景一致，**无分割线**，与内容区无缝衔接。精简布局。

```
┌─────────────────────────────┐
│  ☰   ATLAS        📎  [Z]  │
└─────────────────────────────┘
```

去掉对话标题（空间不够）、去掉 ⋮ 更多菜单（通过长按对话或其他方式触发）。☰ 点击弹出侧栏覆盖层。

## 六、首页（新对话状态）

首页是用户新建对话或首次进入时看到的界面，对标 Gemini 首页的极简风格。

### 6.1 桌面端首页

整体垂直居中偏上（约 40% 位置），大面积留白。

```
                    ✦ Hi {用户名}
              需要我帮你处理什么数据？

          ┌─────────────────────────────┐
          │  直接提问，或上传文件开始分析...  │
          │  + 上传                  发送 → │
          └─────────────────────────────┘

     「出纳模版」「会计模版」「HR 中心」「数据分析」
```

| 元素 | 规格 |
|------|------|
| ✦ 图标 | ATLAS 品牌图标（四角星），品牌色，16px |
| "Hi {用户名}" | 常规字重，18px，灰色 |
| "需要我帮你处理什么数据？" | 加粗，28px，黑色，主标题 |
| 对话框 | 白色圆角卡片（radius 24px），带浅灰边框，宽度 max-width 680px |
| 对话框 placeholder | "直接提问，或上传 Excel/CSV 文件开始分析..." |
| 对话框底部左侧 | + 上传按钮（点击弹出文件选择器） |
| 对话框底部右侧 | 发送按钮（输入内容后激活，品牌色） |
| 快捷标签 | 四个胶囊形按钮，白色背景，浅灰边框，水平排列，间距 12px |

### 6.2 快捷标签行为

| 标签 | 点击后行为 |
|------|----------|
| 出纳模版 | 对话框自动填入"我要使用出纳模版"并发送，AI 回复可用模板列表供选择 |
| 会计模版 | 对话框自动填入"我要使用会计模版"并发送，AI 回复可用模板列表供选择 |
| HR 中心 | 对话框自动填入"我要使用 HR 功能"并发送，AI 回复 HR 功能选项（工资条/考勤） |
| 数据分析 | 对话框自动填入"我要进行数据分析"并发送，AI 回复引导上传文件 |

### 6.3 手机端首页

1:1 对标 Gemini 移动端。欢迎词和对话框居中，快捷标签横向可滚动（超出屏幕宽度时）。对话框宽度跟随屏幕宽度，左右留 16px 边距。

## 七、对话流界面

### 7.1 消息布局

对话流采用 Gemini 的布局方式：

**用户消息**：右对齐，灰色圆角气泡背景（#e8eaed），深色文字。

**AI 回复**：左对齐，无背景色（直接显示在白色/浅灰页面上），左侧带 ✦ ATLAS 图标。回复内容支持 Markdown 渲染（加粗、列表、表格、链接）。

**对话框**：固定在页面底部，白色卡片样式，与首页对话框一致。对话进行中时 placeholder 变为"继续提问..."。

### 7.2 AI 回复操作按钮栏

每条 AI 回复底部显示操作按钮，默认灰色小图标，hover 时高亮。

```
  👍   👎   🔄   📋   ⋮                    < 2/2 >
```

| 按钮 | 功能 | 说明 |
|------|------|------|
| 👍 有用 | 正向反馈 | 点击后图标填充，记录反馈 |
| 👎 不准确 | 负向反馈 | 点击后弹出选项："数据有误""格式不对""不是我要的" |
| 🔄 重做 | 重新生成 | 重新生成回复（含文件），保留历史版本 |
| 📋 复制 | 复制文本 | 复制回复文字内容到剪贴板，显示"已复制"提示 |
| ⋮ 更多 | 扩展选项 | 弹出菜单：「更详细」「更简洁」「再试一次」 |
| < 2/2 > | 版本翻页 | 重做后出现，在多个生成版本之间切换 |

### 7.3 关键数字高亮

AI 回复中的关键数据自动高亮显示，使用品牌色加粗：

> 本月总销售额 **¥128,450.00**，环比增长 **+12.3%**，其中自营占比 **67.2%**。

金额、百分比、数量等数字使用品牌色（蓝色系）加粗，让财务人员一眼抓到关键数据。

### 7.4 文件下载卡片

AI 生成文件后，在对话流中显示文件卡片：

```
┌─────────────────────────────────────┐
│ 📊 销售汇总报表_202603.xlsx          │
│    XLSX · 已生成                     │
│                      [预览]  [下载]  │
└─────────────────────────────────────┘
```

| 元素 | 说明 |
|------|------|
| 文件图标 | 根据文件类型显示不同图标（📊 Excel / 📄 CSV / 📋 PDF） |
| 文件名 | 加粗，单行截断 |
| 格式标签 | 灰色小字，显示文件格式和状态 |
| 预览按钮 | 点击弹出全屏预览覆盖层 |
| 下载按钮 | 品牌色，点击直接下载文件 |

### 7.5 重型表格摘要卡片

当 AI 回复包含大表格（超过 10 行）时，对话流中不直接展开完整表格，而是显示摘要卡片：

```
┌─────────────────────────────────────┐
│ 📊 销售数据汇总                      │
│                                     │
│  门店    │ 销售额     │ 环比        │
│  ────────┼───────────┼────────     │
│  旗舰店  │ ¥45,200   │ +8.3%      │
│  二店    │ ¥32,100   │ -2.1%      │
│  三店    │ ¥28,900   │ +15.7%     │
│  ... 共 47 行                       │
│                                     │
│              [查看完整表格]  [下载]    │
└─────────────────────────────────────┘
```

对话流中只显示前 3-5 行预览，点击"查看完整表格"弹出全屏预览。

### 7.6 处理状态动态显示

AI 处理文件时，对话流中显示动态状态条，让用户看到进度：

```
  ✦ ⟳ 正在读取文件结构...
  ✦ ⟳ 识别到 12 个字段，3,847 行数据
  ✦ ⟳ 正在分析销售趋势...
  ✦ ✓ 分析完成
```

状态条使用打字机效果逐行出现，⟳ 为旋转动画图标，✓ 为完成标记（绿色）。

## 八、文件管理

### 8.1 文件上传

**上传入口**：对话框底部左侧的 + 按钮，点击弹出系统文件选择器。支持多文件同时上传。

**上传中状态**：文件卡片显示在对话框上方（跟 Gemini 一致），显示文件名 + 上传进度条。

```
  ┌──────────────────────────┐
  │ 📄 订单数据_3月.xlsx  ██░░ 65%  ✕ │
  │ 📄 退款明细.csv       ████ 100% ✓ │
  └──────────────────────────┘
  ┌─────────────────────────────┐
  │  继续提问...                  │
  │  + 上传                 发送 → │
  └─────────────────────────────┘
```

**手机端上传**：+ 按钮点击后弹出底部 action sheet：「从相册选择」「从文件选择」「拍照」。

### 8.2 回形针文件面板（📎）

位置：顶栏右侧，回形针图标 + 数量角标。

**悬浮预览**：鼠标 hover 在 📎 图标上时，显示悬浮卡片，列出当前对话的所有文件（最多显示 5 个，超出显示"+N 个文件"）。

**点击展开**：点击 📎 图标，从右侧滑出文件面板（宽度 320px），覆盖在主内容区上方。

文件面板内容：

```
┌─────────────────────────┐
│  此对话的文件        [✕]  │
├─────────────────────────┤
│ 📄 订单数据_3月.xlsx      │
│    ✓ 已解析 · 3,847 行   │
│                          │
│ 📄 退款明细.csv           │
│    ⟳ 解析中...           │
│                          │
│ 📄 库存表.xlsx            │
│    ✕ 解析失败 [重试]      │
└─────────────────────────┘
```

每个文件显示：文件名、处理状态标签（✓ 已解析 / ⟳ 解析中 / ✕ 解析失败）、数据量（行数）。解析失败的文件提供"重试"按钮。

## 十、设置覆盖层

点击侧栏底部 ⚙️ 设置后，弹出覆盖层（居中弹窗，宽度 640px，高度 480px）。

| Tab | 内容 |
|-----|------|
| 个人信息 | 头像上传/修改、昵称编辑、账户信息 |

### 10.2 库 Tab 详情

库作为设置覆盖层的一个 tab，展示用户历史生成的文件和报表：

每个文件显示：文件名、生成时间、来源对话名称、预览/下载按钮。支持按时间/类型筛选。

## 十二、错误与异常状态

所有错误提示在对话流内以红色提示卡片形式显示，不使用弹窗打断用户。

### 12.1 错误卡片样式

```
┌─────────────────────────────────────┐
│ ⚠️ 文件解析失败                       │
│                                     │
│ "订单数据.xlsx"格式异常，无法读取。     │
│ 请检查文件是否损坏，或尝试另存为        │
│ .csv 格式后重新上传。                  │
│                          [重新上传]   │
└─────────────────────────────────────┘
```

卡片左侧带红色竖条，⚠️ 图标，提供明确的错误原因和操作建议。

### 12.2 错误类型

| 错误类型 | 提示内容 | 操作按钮 |
|---------|---------|---------|
| 文件解析失败 | 说明失败原因 + 建议 | [重新上传] |
| AI 处理超时 | "处理时间较长，请稍候" | [重试] |
| 网络断开 | "网络连接中断，请检查网络" | [重试] |
| 文件过大 | "文件超过大小限制（16MB）" | [压缩后重试] |

## 十三、对话标题自动生成

用户发送第一条消息后，系统根据对话内容自动生成标题。标题显示在顶栏居中位置和侧栏历史列表中。

生成规则：
- 基于用户第一条消息和 AI 第一条回复的内容，由 AI 生成简短标题（8-15 个中文字）
- 上传文件时，标题包含文件相关信息（如"3月销售数据分析"）
- 使用模板时，标题包含模板名称（如"出纳日报表生成"）

用户可通过顶栏 ⋮ 菜单的"重命名"手动修改标题。
