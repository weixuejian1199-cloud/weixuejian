## 一、验收指令（通用模板）

> **管理链路说明**：大魏只对 林深（CTO·工程全局）和 启序（CEO·公司全局）下达指令。林深负责调度所有工程团队与专家，向大魏汇报结论。

大魏只用自然语言下达指令，无需文件路径、无需指定人员、无需任何技术细节。

**示例指令（均有效）**：
- "林深，验收会计页面"
- "林深，做运营页面UI审查"
- "林深，财务板块全面评审"
- "林深，客服页面看一下"

林深收到后自动完成：
1. **意图识别** → 判断是验收/审查/评审，确定目标板块
2. **文件定位** → 自动找到对应原型文件（见第六节文件索引）
3. **团队匹配** → 按第四节规则调度对应人员
4. **三层展开** → 按第三节框架执行（修复核查→专业完备→企业级缺口）
5. **结论汇总** → 向大魏输出验收报告（见第五节格式）

**大魏的沟通方式永远是自然语言，技术翻译由林深负责。**

## 二、验收触发条件

| 触发场景 | 自动启动 |
|---------|---------|
| 页面原型修改后 | UI验收团队（韩月+小金+小畅+前端） |
| 功能模块新增/改版 | 对应业务负责人+专家组 |
| 文档更新 | 文档一致性核查（与原型/接口/需求三方对齐） |
| 跨板块影响评估 | CTO林深 + 受影响板块负责人联合验收 |
| 企业级能力审查 | CEO启序 + 各板块负责人全员 |

## 三、验收维度标准

### 3.1 页面验收（UI/原型）

| 维度 | 标准 | 负责人 |
|------|------|--------|
| 修复核查 | 上轮所有问题逐一对号入座，不遗漏 | 韩月 |
| 视觉规范 | 符合 `07-UI设计规范.md` CSS变量/间距/字体规范 | 小畅 |
| 专业准确性 | 数据、流程、术语符合行业标准 | 对应业务专家 |
| 交互完整性 | 所有按钮/Tab/Modal有响应，无死角 | 小金 |
| 移动端适配 | 关键视图不破版（min-width兼容） | 韩月 |
| 无障碍基线 | 关键按钮有title/aria-label | 韩月 |

### 3.2 功能验收（业务逻辑）

| 维度 | 标准 | 负责人 |
|------|------|--------|
| 流程完整性 | 端到端流程无断点（草稿→审核→完成） | 业务负责人 |
| 内控合规 | 权限隔离、制单审核分离等原则落地 | 四大背景专家 |
| 数据准确性 | Mock数据符合真实业务逻辑，无悖论 | 吴川 |
| AI能力匹配 | AI介入点设计合理，置信度/确认机制存在 | 会计Agent |
| 异常处理 | 边界情况有提示/兜底，不白屏 | 吴川 |

### 3.3 文档一致性验收

| 对象 | 核查点 |
|------|--------|
| 原型 vs 需求文档 | 功能点与 `09/11/13/15/16` 等升级方案一致 |
| 原型 vs 接口契约 | 字段名、数据结构与 `08-前后端接口契约.md` 对齐 |
| 原型 vs UI规范 | 颜色/字号/间距与 `07-UI设计规范.md` 一致 |
| 文档 vs 文档 | 跨文档引用无矛盾（如权限设计在多处描述须一致） |

### 3.4 企业级能力验收（每季度 / 大版本必做）

以下为启序科技产品必须支持的企业级能力，每次大版本验收时逐项核查是否已覆盖或有路线图：

| 能力 | 说明 | 当前状态 |
|------|------|---------|
| 多公司主体管理 | 同一平台管理多个法人主体，账套独立 | 待规划 |
| 多账套切换 | 不同行业/公司账套一键切换，数据隔离 | 待规划 |
| 多行业科目模板 | 电商/餐饮/制造等行业预设科目一键启用 | 部分（电商已有） |
| RPA银行卡绑定 | 银行流水自动同步，无需手工导入 | 参见 `14/18-RPA方案` |
| 财务管理员角色 | 独立于会计/出纳的系统配置角色 | 待规划 |
| 老板免登录订阅 | CEO通过微信/邮件收经营日报，无需每天登录 | 待规划 |
| 系统级审计日志 | 全平台操作可追溯，不限于财务板块 | 待规划 |
| 权限中心统一管理 | super_admin统一分配各板块权限 | 部分（v3-02已有） |

## 五、验收输出标准

每次验收结束，负责人须输出：

```
【验收结论】
板块：xxx
验收人：xxx
日期：yyyy-mm-dd

已解决：
- [问题] → [验证方式] → ✅ 通过

新发现缺口：
- [缺口描述] → 优先级：P0/P1/P2 → 建议处理时间

企业级能力缺口（如有）：
- [能力] → 当前状态 → 路线图建议

总结论：可上线 / 需修复后复验 / 打回重做
```
