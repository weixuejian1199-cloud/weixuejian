## 三、P0问题清单（不解决就不算客服系统）

### P0-1. 工单生命周期管理

**问题**：只有"会话"没有"工单"，会话结束后数据去哪了？无法追踪、无法审计、无法复盘。

**五方意见**：
- 客服总监：这不叫工单管理，这叫工单入口。真正的工单需要详情页、流转记录、SLA倒计时
- 独立站专家：Gorgias的核心就是把每个客户互动变成一个可追踪的ticket
- 小微店主：我现在全靠记忆和便利贴追踪没处理完的事，工单帮我记着

**升级方案**：
- 工单状态机：新建→已分配→处理中→待客户回复→已解决→已关闭→重新打开
- 每个状态转换写入审计日志，SLA时钟在"待客户回复"时暂停
- 工单详情页：完整对话记录 + AI判断记录 + 流转时间线 + 关联订单
- 支持工单合并（同一客户多次咨询同一问题）
- 工单模板：退货/换货/投诉/咨询/过敏预设模板

**对标**：Zendesk Ticket Lifecycle + 京东工单系统

### P0-2. SLA时效管理与自动升级

**问题**：天猫3分钟首响、抖店5分钟首响，超时扣分罚款。当前连SLA倒计时都没有。

**五方意见**：
- 护肤品主管：这是日常运维的生命线，超时一次扣0.1分，月底不达标直接限流
- 客服总监：没有SLA等于没有管理，这是客服运营的水电煤
- 独立站专家：Zendesk/Freshdesk的核心就是SLA

**升级方案**：
- 3级SLA策略：VIP客户（首响30s/解决2h）→ 普通客户（首响60s/解决4h）→ 低优先级（首响5min/解决24h）
- 渠道级SLA：天猫3分钟、抖店5分钟、小程序10分钟（可配置）
- 超时三级提醒：黄灯预警(80%)→红灯告警(100%)→自动升级主管(120%)
- 会话列表每条显示SLA倒计时进度条，超时红色高亮置顶
- SLA达标率实时显示在Dashboard

**对标**：Zendesk SLA Policies + Salesforce Entitlement Process

### P0-3. 智能路由与自动分配

**问题**：3个"待接入"会话无人认领是运营灾难。当前只有手动"转接"，没有自动分配逻辑。

**五方意见**：
- 客服总监：按技能/负载/客户等级自动分配是基本功
- 独立站专家：Intercom有Round Robin、Load Balancing、Skill-based Routing

**升级方案**：
- 自动分配规则（可配置）：
  - 按技能标签：护肤品问题→美容顾问，物流问题→物流专员
  - 按负载均衡：当前接待数最少的优先
  - VIP客户→资深坐席
  - 情绪激动→高级坐席
- 转接面板智能推荐："推荐转接：小李（护肤专家，当前负载2/5）"
- 排班表可视化：实时看每人接待量和等待队列

**对标**：Zendesk Omnichannel Routing + Intercom Assignment Rules

### P0-4. 护肤品成分数据库 + AI成分咨询

**问题**：护肤品客服最高频问题——"含不含酒精？""孕妇能用吗？""烟酰胺浓度多少？"完全没有覆盖。

**护肤品主管原话**：
> "每天至少50条成分咨询，新客服培训两周都记不全成分表。"

**升级方案**：
- 建立SKU成分数据库：每个产品完整成分表 + 核心功效成分 + 浓度 + 禁忌人群
- AI成分咨询能力：自动检索成分表，判断成分禁忌（孕妇禁用/常见过敏原）
- 搭配推荐引擎：根据已购产品+肤质标签推荐搭配，一键发送产品卡片
- 客户面板增加肤质档案：肤质类型/过敏史/在用产品/护肤诉求

**对标**：晓多美妆成分知识库 + 乐言肤质测试问卷

### P0-5. 过敏投诉全流程SOP

**问题**：护肤品最高压场景。当前只展示了"退货退款"一个分支，处理不好就是差评+消协投诉+赔偿。

**护肤品主管详述**：过敏处理至少6步：
1. 确认过敏症状+拍照留证
2. 建议停用+就医指引
3. 发起退货，同时标记批次号（做质检追溯）
4. 48小时内跟进客户就医情况
5. 根据严重程度决定是否额外补偿
6. 通知品控部门（产品批次质检）

**升级方案**：
- 过敏工单自动触发6步SOP流转，每步有时间节点提醒
- 批次号自动关联品控预警系统
- 补偿决策矩阵：轻微过敏(退款)→中度过敏(退款+补偿券)→严重过敏(退款+现金补偿+升级处理)
- 工单内嵌"客户就医情况跟进"字段

**小微店主原话**：
> "客户说过敏了怎么办——这种关键词必须手机立刻弹通知，5分钟内我必须亲自处理。"

### P0-6. 实时客服Dashboard

**问题**：工作概览只是侧边栏一个链接，底部只有一行小字。日均10万工单的客服中心至少需要30+个KPI。

**升级方案**：
- 实时仪表盘核心指标：
  - 排队等待人数/平均等待时间
  - 各坐席当前负载
  - SLA达标率（整体+分渠道）
  - AI处理率趋势图
  - 情绪预警汇总
  - 退货率趋势/退货原因分布
- 坐席绩效对比：接待量/满意度/解决率排名
- 大屏模式：适合投屏到客服中心
- 小微店主简化版：今天几单/退了几单/几个人在等回复（一眼看到）

**对标**：Zendesk Explore Real-time Dashboard + 京东客服大屏

### P0-7. RAG检索链路设计

**问题**：AI技术专家指出——当前最大技术空白。RAG没有详细设计，准确率无从保证。

**AI专家详述**：
- 纯向量检索、无reranker、无置信度门槛：准确率55-65%
- 混合检索 + reranker + 门槛：准确率80-90%
- 加上FAQ精确匹配层：准确率90%+

**升级方案**：
- 三层检索架构：
  1. FAQ精确匹配（hash/ES exact match）→ 命中直接返回
  2. 向量+BM25混合检索 → reranker排序
  3. 均未命中 → fallback话术 + 转人工
- Embedding：百炼text-embedding-v3（中文优化）
- Chunk策略：按FAQ条目切分，保留品类/场景metadata
- 检索置信度阈值：0.75以下拒绝回答，走fallback

### P0-8. 幻觉防护（对话层）

**问题**：当前只守住了"操作层"（工单人审+硬规则），但"对话层"完全裸奔。AI可能编造成分表、虚构退货政策、承诺不存在的优惠。

**AI专家详述**——需要补齐的6层防线：

| 防线层级 | 说明 | 当前状态 |
|----------|------|----------|
| L1 Prompt约束 | System prompt明确"只基于提供的信息回答" | 缺失 |
| L2 RAG Grounding | 回答必须引用知识库来源 | 缺失 |
| L3 检索置信度门槛 | 低于阈值拒绝回答 | 缺失 |
| L4 输出过滤 | 禁言词/PII脱敏/格式校验 | 缺失 |
| L5 人工确认 | 工单操作人审 | **已有** |
| L6 硬规则覆盖 | 业务规则不经LLM | **已有** |

**升级方案**：
- L1：System prompt增加"你只能基于以下信息回答，不确定时说'我需要确认'"
- L2：每条AI回复标注来源（知识库/工具返回/AI生成），AI生成部分标记"仅供参考"
- L3：RAG检索分<0.75时输出标准话术"正在为您转接专员"并自动创工单
- L4：禁言词正则（"保证赔偿"/"我承诺"等）+ PII脱敏（身份证/银行卡正则替换***）

### P0-9. 新手引导与傻瓜模式

**问题**：7个导航模块对个人店主太多，知识库录入和AI规则配置是学习门槛。

**小微店主原话**：
> "我是卖货的不是搞技术的。知识库、工单系统、数据报表这些词看着就头疼。最好给我一个傻瓜模式。"

**升级方案**：
- 首次登录3步引导：欢迎→试处理一个AI建议→查看今日待办
- 小白/专业模式toggle：小白模式隐藏高级功能，只显示"聊天+AI建议+客户信息"
- 护肤品行业模板：一键导入常见问答（物流/退换/成分/使用方法），10分钟跑起来
- 术语翻译：工单→待处理事项、SLA→回复时限、质检→服务评分

### AI角色定位原则

| 操作类型 | AI角色 | 理由 |
|---------|--------|------|
| FAQ/物流查询 | AI主导，无需人工 | 确定性高，0风险 |
| 成分咨询 | AI主导，知识库grounding | 数据库查询，准确率可保证 |
| 退货审核 | AI判断，人工确认 | 涉及退款，Phase 1必须人审 |
| 过敏投诉 | AI辅助，人工主导 | 高敏感场景，AI做SOP提醒和数据准备 |
| 情绪升级 | AI检测，自动转人工 | 情绪问题必须真人处理 |
| 砍价/议价 | 人工主导 | AI做不到"看人下菜碟" |
| 补偿决策 | 人工主导，AI建议 | 金额决策权在人 |

### AI风控机制

1. **六层幻觉防护**：Prompt约束→RAG Grounding→置信度门槛→输出过滤→人工确认→硬规则覆盖
2. **风险闸门硬编码**：退款必须人审、情绪激动立即升级、欺诈立即升级、不确定转人工
3. **对话轮次兜底**：超过8轮未解决，无论情绪检测结果如何，自动升级人工
4. **回调防死循环**：processedByHuman标记 + Redis 10分钟限流锁
5. **渐进放权闸门**：陪练连续稳定2周+关键指标不波动→Phase 2可开放低风险自动处理
