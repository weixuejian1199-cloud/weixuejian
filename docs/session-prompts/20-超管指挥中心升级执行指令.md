## 核心洞察（必读）

### 当前问题
超管指挥中心(v3-02)目前只是一个"看数据的仪表盘"——老板看完数据还是要切到各个板块去操作。
工具配置(v3-07)目前只是"配置Agent的后台"——老板在这里只能配置，不能使用。

### 应该是什么
**超管指挥中心 = 作战指挥台**
- 不只是看，更是**做**——老板在这一个页面就能跨财务/客服/运营下命令
- "天猫退货率涨了，帮我查原因并通知客服负责人" → AI跨客服+运营板块执行
- "这个月现金流够不够发工资？" → AI跨财务板块回答
- "给我出一份本周经营周报，发飞书群" → AI聚合所有板块数据生成

**工具中心 = 武器库/能力市场**
- 不是配置，而是**使用和组合**——就像iPhone的App Store
- 每个工具是一个可直接调用的AI能力（选品助手、标题生成器、竞品雷达、利润计算器...）
- 老板/运营/客服各角色看到不同的工具推荐
- 工具之间可以串联（选品→定价→详情页生成→上架 一条龙）

### 这两个板块的关系
```
超管指挥中心 = 大脑（决策+指挥+全局视野）
工具中心     = 武器库（能力+工具+一键调用）
财务/客服/运营 = 执行部队（各司其职的专业战场）
```

老板的工作流应该是：
1. 打开指挥中心 → 看到今天需要关注什么（AI晨报）
2. 发现问题 → 直接在指挥中心对话下命令 → AI跨板块执行
3. 需要具体工具 → 从工具中心调用（或AI自动调用）
4. 需要深入某个板块 → 再跳转到专业工作台

### 设计原则（不可违反）

**超管指挥中心特殊原则：**
- 这是老板每天第一个打开的页面——信息密度要高但不能乱
- AI是主角——进来第一眼看到的应该是"AI帮你准备好的今日简报"，不是空白对话框
- 可操作——看到问题就能在这里下命令，不需要跳走
- 全局视野——财务/客服/运营的关键指标都要出现，但只出现异常和趋势，不堆数据

**工具中心特殊原则：**
- 用户来这里是"用工具"的，不是"配置工具"的——使用优先于管理
- 每个工具要能在页面内直接使用，不是跳到外部
- 工具之间要能串联，不是一个个孤岛
- AI能自动推荐"你现在可能需要这个工具"

**⛔ 文件隔离规则（并行对话安全）：**
- **只允许修改 v3-02-super-admin.html 和 v3-07-admin-config.html**，不得修改其他v3-0x文件
- 其他页面的导航更新由各自的对话负责，不要越界
- 如果需要参考其他页面的代码，只读不写

**通用原则：**
- 视觉风格：黑白极简，100分不可改
- CSS变量集与其他页面完全一致
- 顶部模块切换导航已移除（岗位无权限跨模块）；super_admin 通过 sidebar"指挥中心"入口在岗位视角与 v3-02 之间双向跳转
- 数字monospace、min-width:1024px、sidebar滚动条4px
- 新手引导弹窗 + 小白/专业模式 toggle
- 每个视图都有AI对话面板

### 质量标准
- 七位Agent评审加权平均目标：≥90分
- 两个页面都必须有多个可切换视图
- 超管页面要能"10秒抓到今天最重要的3件事"
- 工具中心的工具要能在页面内直接使用（至少模拟交互）
- 数据要真实合理（护肤品电商场景）
