## 三、三方共识的P0问题清单

以下8项是三位CFO一致认为"不解决就不能作为财务系统使用"的问题：

### P0-1. 会计科目体系缺失

**问题**：没有科目体系，凭证无法创建，报表无法生成——这是财务系统的"数据库Schema"。

**三方意见**：
- 上市CFO：这是P0中的P0，没有科目就没有财务系统
- 电商CFO：需要电商行业专属科目模板（含渠道辅助核算维度）
- 小白财务：科目是什么？需要AI帮我自动设置

**升级方案**：
- 预设"电商行业标准科目模板"（参照《企业会计准则》+ 电商场景定制）
- 首次使用引导：AI根据用户描述的业务（品类、渠道、规模）自动推荐科目体系
- 支持自定义扩展，但预设模板覆盖90%场景
- 辅助核算维度：渠道(必选) + 客商 + 项目（可选）

**对标**：金蝶云星辰「基础设置 → 会计科目」+ AI智能推荐

### P0-2. 凭证全生命周期不完整

**问题**：当前只有凭证列表，缺少制单→审核→记账→反记账→红字冲销的完整链路。

**三方意见**：
- 上市CFO：中国会计准则要求凭证一经记账不得随意修改，只能红字冲销
- 电商CFO：需要从平台结算单自动生成凭证的能力
- 小白财务：希望AI自动记账，但确认时不要给我看"借""贷"

**升级方案**：
- 凭证状态机：草稿 → 待审核 → 已审核 → 已记账（不可逆向修改，只能红冲）
- 审核人不能是制单人（内控自动校验）
- AI自动生成凭证 + 自然语言确认卡片（默认显示"天猫回款28600已到账"，可展开看专业分录）
- 凭证来源标注（AI自动 / 手工录入 / 平台导入）

**对标**：用友好会计凭证管理 + AI辅助制单

### P0-3. 发票管理模块空白

**问题**：电商企业月开票/收票量大，发票是税务核心凭据，也是AI价值最大的切入点。

**三方意见**：
- 上市CFO：票财税一体化是趋势，发票管理不可缺
- 电商CFO：进项发票（原料/包材/物流）的认证直接影响税负
- 小白财务：拍照就能入账是刚需

**升级方案**：
- 进项发票：OCR拍照识别 → 自动录入 → 关联采购单 → 认证提醒
- 销项发票：与订单关联 → 批量开具 → 台账查询
- 发票-采购单-付款单三单匹配
- AI自动计算进销项差额、预测税负率

**对标**：金蝶云星辰「发票管理」+ OCR智能识别

### P0-4. 平台结算对账引擎缺失

**问题**：电商财务每月最耗时的工作。当前银行对账只做了传统银企对账，缺少平台结算单对账。

**电商CFO详述**：
- 天猫结算单金额 ≠ 订单销售额（扣佣金/退款扣回/补贴到账）
- 每个平台结算规则不同，差异来源复杂
- 目前靠Excel手动对，一个平台要1-2天

**升级方案**：
- 新增"平台对账"模块（与银行对账平级）
- 支持各平台结算单CSV/API导入
- 三方自动匹配：平台结算金额 = 订单销售额 - 平台扣点 - 退款扣回 + 补贴
- AI自动标注差异项并给出可能原因
- 结算日历：自动标注各平台预计/实际到账日

**效率提升**：每月节省3-5个工作日，准确率85%→99%

**小白财务原话**："如果能做到说一句话就帮我对完天猫这个月的回款，光凭这一个功能就愿意换掉Excel。"

### P0-5. SKU级成本核算空白

**问题**：当前"营业成本"只是一个黑盒数字（且净利率43.3%严重偏高，说明成本口径不全）。

**电商CFO详述**：护肤品SKU成本构成：
- 原料成本（按BOM配方）
- 包材成本（瓶器+纸盒+说明书+封条+快递盒+填充物）
- 灌装人工/代工费
- 仓储分摊
- 物流快递费（按单/按重量/按地区）
- 平台佣金/技术服务费（按渠道不同）
- 推广费用（按渠道/按campaign）

**升级方案**：
- 建立SKU成本BOM模板
- 支持按月/按批次更新成本
- AI自动按BOM计算每个SKU完整成本和毛利
- 区分引流款/爆款/利润款的ROI

**效率提升**：从"说不清哪个产品赚钱"到实时可查，决策响应从周→秒

### P0-6. 退货退款财务流程零覆盖

**问题**：护肤品退货率5-20%（抖店可达20%+），涉及收入冲减、库存回收、退运费归集、平台退款手续费。

**升级方案**：
- 退货触发自动红冲凭证（冲减收入+冲减成本）
- 退回商品自动入库并标注品质状态（可售/残次/报废→不同科目）
- 退款手续费自动归集到财务费用
- AI统计退货率趋势并预警异常

**效率提升**：退货凭证处理每笔10分钟→自动化，月末减少1-2天

### P0-7. 财务专属导航入口

**问题**：PC端Header原来没有"财务"入口，财务角色找不到工作台。

**状态**：已解决。导航已拆分为"会计"+"出纳"两个独立入口。

### P0-8. 新手引导与术语翻译

**问题**：专业术语（凭证/总账/科目余额表/期末处理/未达账项）让小白用户看不懂、不敢用。

**升级方案**：
- 小白模式术语映射：凭证→记账单、总账→账目汇总、科目余额表→各项目余额、期末处理→月末关账、未达账项→待核实款项
- 首次登录3步引导：欢迎→试记一笔账→查看今日待办
- AI确认卡片用自然语言（默认折叠专业分录）

## 六、AI差异化能力矩阵

这是我们超越金蝶/用友的核心武器：

| AI能力 | 金蝶能做？ | 用友能做？ | 我们的实现 | 效率提升 |
|--------|----------|----------|-----------|---------|
| 对话式凭证录入 | 不能 | 不能 | "记一笔：天猫回款28600"→自动拆科目 | 80%+ |
| 银行流水→批量凭证 | 规则匹配 | 规则匹配 | AI识别+自动匹配科目+批量生成 | 80%+ |
| 跨模块关联分析 | 不能 | 不能 | "为什么毛利率下降3个点"→AI归因 | 60%+ |
| 异常主动预警 | 预设规则 | 预设规则 | AI主动发现异常并推送（不需配规则） | 40%+ |
| 自然语言报表 | 不能 | 不能 | "出一份3月经营简报给老板"→PDF | 90%+ |
| 平台回款自动对账 | 不能 | 不能 | AI三方匹配+差异标注+原因分析 | 3天→10分钟 |
| 发票OCR入账 | 第三方集成 | 第三方集成 | 拍照→识别→关联→入账一条龙 | 90%+ |
| 期末自动结转 | 按钮触发 | 按钮触发 | "执行本月期末结转"→全自动 | 95%+ |
| 催收建议 | 不能 | 不能 | AI识别逾期→生成催收方案 | 40%+ |
| 账务健康检查 | 不能 | 不能 | 每日自动检查借贷平衡/异常金额 | 事后→实时 |

### AI角色定位原则

| 操作类型 | AI角色 | 理由 |
|---------|--------|------|
| 凭证生成 | 主导生成，人工确认 | 效率巨大，但科目错误影响报表 |
| 凭证审核 | 辅助检查，人工审核 | 审核权必须在人（内控要求） |
| 银行对账 | 主导匹配，人工确认差异 | 自动匹配95%+准确率 |
| 报表出具 | AI主导 | 数据正确则报表确定性生成 |
| 报表分析 | AI主导分析，人工判断 | AI擅长发现异常，决策靠人 |
| 期末结转 | AI主导执行，人工触发确认 | 计算逻辑确定，时点需人决定 |
| 税务申报 | AI辅助计算，人工确认提交 | 合规风险高，不能全自动 |

### AI风控机制

1. **金额阈值控制**：超设定金额的AI操作必须二次确认
2. **操作可回滚**：AI生成的凭证能一键撤回
3. **AI决策日志**：记录每次操作的推理过程，审计可查
4. **置信度显示**：低于阈值强制人工复核

## 九、权限边界设计（会计 vs 出纳）

### 会计可以 / 不可以

| 可以 | 不可以 |
|------|--------|
| 做凭证、审凭证 | 查看银行真实余额明细 |
| 查账、出报表 | 执行付款操作 |
| 做期末处理 | 修改银行对账结果 |
| 管理发票 | 登记收付款 |
| 设置科目 | 管理票据 |

### 出纳可以 / 不可以

| 可以 | 不可以 |
|------|--------|
| 登记收付款 | 做凭证、审凭证 |
| 做银行对账 | 修改总账数据 |
| 出资金报表 | 出利润表等财务报表 |
| 管理票据 | 设置会计科目 |
| 执行付款 | 审批付款（需会计/老板审批） |
